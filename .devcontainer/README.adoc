= .devcontainer Directory
:description: Development container configuration for paint-type

This directory contains the **development container** configuration for paint-type,
used by VS Code Dev Containers, GitHub Codespaces, and Gitpod.

== Purpose

Provides a consistent, reproducible development environment that matches
the production build environment. This ensures:

* All developers use the same toolchain versions
* Builds in dev match builds in CI
* Security patches are consistently applied

As a Level 1 directory (`.devcontainer/`), it supports the overall
project structure.

== Contents by Functional Grouping

=== Configuration

* `devcontainer.json` -- VS Code/GitHub Codespaces configuration
* `Containerfile` -- Development container image definition

=== Documentation

* `README.adoc` -- This file
* link:../CONTAINER-GUIDE.adoc[Container Architecture Guide] -- Complete guide to all container configurations

== Purpose of Each Component

Each file in this directory serves a specific purpose:

*Configuration files* define build, test, and deployment parameters for the development environment.
*Other files* support directory operations.

== Development vs Production

| Aspect | This Container | Production Container |
|--------|----------------|---------------------|
| Purpose | Development | Deployment |
| Location | `.devcontainer/Containerfile` | `Containerfile` (root) |
| Tools | Full dev stack | Minimal runtime |

See link:../CONTAINER-GUIDE.adoc[Container Architecture Guide] for complete details.

== Quick Start

Build the development container:

[source,bash]
----
podman build -t paint-type-dev -f .devcontainer/Containerfile .
----

Run with workspace mounted:

[source,bash]
----
podman run --rm -it -v $(pwd):/workspaces/paint-type paint-type-dev
----
